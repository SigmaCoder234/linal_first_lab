### 1. Постановка задачи
Цель работы — реализовать набор функций для решения СЛАУ вида $Ax=b$ и провести сравнительный анализ методов по времени работы и точности.

В рамках работы необходимо:
* реализовать метод Гаусса без выбора ведущего элемента;
* реализовать метод Гаусса с выбором главного элемента по столбцу;
* реализовать LU-разложение матрицы;
* реализовать прямую и обратную подстановки для решения треугольных систем;
* исследовать влияние размера системы и количества правых частей на время решения;
* проверить точность методов на матрицах Гильберта.

### 2. Описание реализованных алгоритмов

#### 2.1 Классический метод Гаусса (без выбора главного элемента)
Процесс решения базируется на последовательном выполнении двух фаз: приведения матрицы к треугольному виду и вычисления неизвестных.
* **Этап исключения (прямой ход).** На каждой $k$-й итерации происходит обнуление поддиагональных элементов в рабочем столбце. Для любой строки с индексом $i>k$ рассчитывается коэффициент пропорциональности $m_{ik}=\frac{A_{ik}}{A_{kk}}$ (где $j=k,k+1,...,n-1$). Вслед за этим элементы текущей строки и соответствующий компонент правой части пересчитываются по правилу: $A_{ij}\leftarrow A_{ij}-m_{ik}A_{kj}$ и $b_{i}\leftarrow b_{i}-m_{ik}b_{k}$. В результате этих преобразований исходная матрица принимает верхнетреугольный вид.
* **Этап нахождения корней (обратный ход).** Полученная упрощенная система разрешается в обратном порядке. Значения вектора $x$ вычисляются последовательно, начиная с последнего уравнения и двигаясь к первому, согласно выражению: $x_{i}=\frac{b_{i}-\sum_{j=i+1}^{n}U_{ij}x_{j}}{U_{ii}},$ для $i=n,n-1,...,1$

#### 2.2 Алгоритм Гаусса с частичным выбором (по столбцу)
Данная модификация отличается от классической версии тем, что перед началом процедуры исключения на каждом $k$-м шаге производится поиск максимального по абсолютной величине значения в нерассмотренной части текущего столбца. Индекс такой строки определяется как:
$p=arg~max_{i\in\{k,...,n-1\}}|A_{ik}|.$
После нахождения максимума, $k$-я и $p$-я строки взаимно переставляются как в самой матрице системы, так и в векторе свободных членов. Дальнейшие шаги прямого хода идентичны базовому методу. Подобная стратегия существенно подавляет накопление погрешностей округления, делая вычислительный процесс более надежным и стабильным.

#### 2.3 LU-разложение
Суть подхода заключается в представлении изначальной матрицы $A$ как произведения двух треугольных матриц: нижней $L$ (с единицами на главной диагонали) и верхней $U$, то есть $A=LU$. 
Расчет элементов этих матриц ведется по следующим соотношениям:

* Компоненты матрицы $U$: $U_{ik}=A_{ik}-\sum_{j=0}^{i-1}L_{ij}U_{jk},$ при $k\ge i$.
* Компоненты матрицы $L$: $L_{ki}=\frac{A_{ki}-\sum_{j=0}^{i-1}L_{kj}U_{ji}}{U_{ii}},$ при $k>i$.

Благодаря такой декомпозиции, решение исходной проблемы $Ax=b$ сводится к последовательному нахождению корней двух более простых подсистем: $Ly=b$ и $Ux=y$.
* **Нахождение промежуточного вектора (прямая подстановка):** Учитывая нижнетреугольную структуру $L$ и условие $l_{ii}=1$, вектор $y$ вычисляется прямым проходом сверху вниз: $y_{i}=\frac{b_{i}-\sum_{j=0}^{i-1}L_{ij}y_{j}}{L_{ii}},$ для $i=1,...,n$.
* **Определение итогового решения (обратная подстановка):** Искомый вектор $x$ рассчитывается обратным проходом по верхнетреугольной матрице $U$: $x_{i}=\frac{y_{i}-\sum_{j=i+1}^{n}U_{ij}x_{j}}{U_{ii}},$ для $i=n,n-1,...,1$

Главная ценность этого алгоритма состоит в том, что ресурсоемкая операция факторизации матрицы $A$ выполняется лишь однажды. В дальнейшем, при изменении вектора $b$, получение нового ответа требует лишь быстрых проходов по двум готовым треугольным подсистемам.

### 3. Метрики оценки

#### 3.1 Евклидова норма
Для вектора $v$ евклидова норма задаётся формулой:

$$
||v||_{2}=\sqrt{\sum_{i=1}^{n}v_{i}^{2}}.
$$
#### 3.2 Вычитание векторов
Для векторов $v1$ и $v2$ вычитание векторов задается формулой:
$result = v1_i - v2_i,$        $i = 1, 2, ..., n$

#### 3.3 Относительная погрешность
Если $\tilde{x}$ — приближённое решение, а $x$ — точное решение, то относительная погрешность вычисляется как:

$$
\epsilon_{nel} = \frac{||\tilde{x}-x||_{2}}{||x||_{2}} = \frac{\sqrt{\sum_{i=1}^{n}(\tilde{x}_{i}-x_{i})^{2}}}{\sqrt{\sum_{i=1}^{n}x_{i}^{2}}}
$$

#### 3.4 Невязка
Невязка показывает, насколько найденное решение удовлетворяет исходной системе:

$$
\rho = ||A\tilde{x}-b||_{2} = \sqrt{\sum_{i=1}^{n}\left(\sum_{j=1}^{n}a_{ij}\tilde{x}_{j}-b_{i}\right)^{2}}
$$

#### 3.5 Время работы
Время работы измерялось с помощью `std::chrono`. Для каждого эксперимента фиксировалось время начала и окончания вычислений, после чего вычислялась разность в секундах.

### 6. Выводы
1. Метод Гаусса и LU-разложение имеют сравнимую асимптотическую сложность на одной системе, однако LU оправдано при решении многих систем с одной и той же матрицей.

2. На эксперименте с $k = 100$ LU оказалось на порядки быстрее повторного решения методом Гаусса, что подтверждает практическое преимущество разложения.

3. Выбор ведущего элемента повышает численную устойчивость и обычно уменьшает ошибку решения.

4. На матрицах Гильберта невязка может оставаться очень маленькой, в то время как относительная ошибка резко растёт. Это демонстрирует влияние плохой обусловленности.

5. При росте размерности матрицы Гильберта точность быстро ухудшается для всех прямых методов, а pivot обеспечивает наилучшую устойчивость среди рассмотренных реализаций.
